## Where to place the banner?

Before we start working on the banner component, we need to decide where to place it. Ideally, you place the banner in the layout component. This way, you can use the banner component across different pages.

```tsx title="src/app/layout.tsx"
import type { Metadata } from 'next';
import { Geist, Geist_Mono } from 'next/font/google';
import './globals.css';
import Header from '@/components/header';
import Footer from '@/components/footer';

const geistSans = Geist({
  variable: '--font-geist-sans',
  subsets: ['latin'],
});

const geistMono = Geist_Mono({
  variable: '--font-geist-mono',
  subsets: ['latin'],
});

export const metadata: Metadata = {
  title: 'Create Next App',
  description: 'Generated by create next app',
};

export default function RootLayout({
  children,
}: Readonly<{
  children: React.ReactNode;
}>) {
  return (
    <html lang="en">
      <body
        className={`${geistSans.variable} ${geistMono.variable} antialiased`}
      >
        // [!code highlight]
        {/* Here? */}
        <div className="flex h-svh flex-col">
          // [!code highlight]
          {/* Or Here? */}
          <Header />
          <main className="container mx-auto grow">{children}</main>
          <Footer />
        </div>
      </body>
    </html>
  );
}
```

We can place at either of the two locations but keeping it inside the `div` with the `h-svh` class helps us avoid the overflow. Let us create our banner component and place it inside the `div` with the `h-svh` class.

## Component API

We can make the banner component quite flexible by allowing the user to configure it as per their needs. We will address the following questions in the API:

- **Closable?** Should the banner allow users to dismiss it?
- **Persistent?** Should the banner persist across page reloads?
- **Layout?** Should the banner account for the height of the header and push the content below it?
- **Rotating Messages?** Should the banner support multiple messages? Should it cycle through multiple messages automatically or on user action?

Let us tackle each of these questions now.

### Closable

We can make the banner closable by adding a close button to it. This would require converting the component to a client component and bringing in state.

```tsx title="src/components/banner.tsx"
'use client'; // [!code ++]

import React from 'react'; // [!code ++]

export default function Banner() {
  const [isVisible, setIsVisible] = React.useState(true); // [!code ++]

  // [!code ++:3]
  if (!isVisible) {
    return null;
  }

  return (
    // [!code highlight:2]
    <div className="flex items-center justify-between gap-2 bg-red-500 px-2 py-0.5">
      <span>Banner</span>
      // [!code ++:3]
      <button className="font-bold" onClick={() => setIsVisible(false)}>
        X
      </button>
    </div>
  );
}
```

We made the following changes:

- We converted the component to a client component.
- We added the `isVisible` state to the component.
- We used the `if` statement to check if the banner should be visible or not.
- We added a close button to the banner.
- We used the `onClick` event to update the `isVisible` state when the close button is clicked.
- We added some styles to the banner to make it look better.

But there is still one problem. If the user refreshes the page, the banner will still be visible. Let us fix that.

### Persistent

In order to make the banner persistent, we need to store the visibility state in localStorage. But we need a key to store the state in localStorage. Just saving as `banner` can cause conflicts. So, we can now introduce a prop called `id` to the component. This is optional and if provided, it will be used as the key to store the state in localStorage and also display the close button.

```tsx title="src/components/banner.tsx"
'use client';

import React from 'react';

// [!code ++:3]
type BannerProps = {
  id?: string;
};

// [!code highlight]
export default function Banner({ id }: BannerProps) {
  const [isVisible, setIsVisible] = React.useState(true);

  if (!isVisible) {
    return null;
  }

  return (
    <div className="flex items-center justify-between gap-2 bg-red-500 px-2 py-0.5">
      <span>Banner</span>
      // [!code highlight:5]
      {id ? (
        <button className="font-bold" onClick={() => setIsVisible(false)}>
          X
        </button>
      ) : null}
    </div>
  );
}
```

This should now not render the close button, because we haven't passed the id from the parent. Let us add the id to the parent and work on the persistence.

Now the close button should render again. Let us add the persistence now.

```tsx title="src/components/banner.tsx"
'use client';

import React from 'react';

type BannerProps = {
  id?: string;
};

export default function Banner({ id }: BannerProps) {
  const [isVisible, setIsVisible] = React.useState(true);
  // [!code ++]
  const localStorageKey = id ? `banner-${id}` : null;

  // [!code ++:5]
  React.useEffect(() => {
    if (localStorageKey) {
      setIsVisible(localStorage.getItem(localStorageKey) !== 'true');
    }
  }, [localStorageKey]);

  if (!isVisible) {
    return null;
  }

  // [!code ++:6]
  const closeBanner = () => {
    if (localStorageKey) {
      localStorage.setItem(localStorageKey, 'true');
    }
    setIsVisible(false);
  };

  return (
    <div className="flex items-center justify-between gap-2 bg-red-500 px-2 py-0.5">
      <span>Banner</span>
      // [!code highlight:2]
      {localStorageKey ? (
        <button className="font-bold" onClick={closeBanner}>
          X
        </button>
      ) : null}
    </div>
  );
}
```

We made the following changes:

- We use the localStorage key to store the visibility state.
- We sync the visibility state with localStorage using the `useEffect` hook.
- We added the `closeBanner` function to close the banner by setting the visibility state to `false` and syncing it with localStorage.
